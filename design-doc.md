# 皮皮虾的 TodoList - 需求文档 v1.0

## 1. 项目概述

### 1.1 项目名称
皮皮虾的 TodoList（愿望清单系统）

### 1.2 项目定位
一个面向 AI Agent 协作的愿望清单管理系统，支持多个龙虾（agents）共同维护和更新任务列表。

### 1.3 核心特点
- **愿望清单式**：不强制时间计划，关注目标而非截止日期
- **动态更新**：目标可以持续演进和细化
- **分类管理**：按主题/方向组织任务
- **多 Agent 协作**：所有龙虾都能更新和维护

## 2. 参考案例分析

### 2.1 todolist-md-clawdbot
**特点**：
- 基于 Markdown 文件存储
- 使用 bot markers 标记任务
- 支持读取、总结、编辑
- 保持任务身份稳定

**可借鉴**：
- Markdown 作为数据格式（简单、可读）
- 任务标记系统
- 版本控制友好

### 2.2 技能交换网站逻辑
**特点**：
- 提交表单 → GitHub Issue
- 审核 → 接受/拒绝
- 动态展示

**可借鉴**：
- 使用 GitHub 作为后端存储
- 简单的提交和审核流程
- 静态网站 + API

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 任务管理
- [ ] 创建任务（标题、描述、分类）
- [ ] 编辑任务
- [ ] 删除任务
- [ ] 标记完成/未完成
- [ ] 任务优先级（P0/P1/P2/P3）

#### 3.1.2 分类管理
- [ ] 预设分类（龙虾+微信、飞书+龙虾、龙虾进化）
- [ ] 自定义分类
- [ ] 分类颜色标记

#### 3.1.3 多 Agent 协作
- [ ] Agent 身份识别（丑丑虾/皮皮虾/乖乖虾/乐乐虾/锥锥虾）
- [ ] 任务更新记录（谁在什么时候更新了什么）
- [ ] 协作日志

### 3.2 界面需求

#### 3.2.1 首页
- 分类卡片展示
- 每个分类显示任务数量和进度
- 快速添加任务入口

#### 3.2.2 分类详情页
- 任务列表（可折叠子任务）
- 筛选（全部/进行中/已完成）
- 排序（优先级/创建时间/更新时间）

#### 3.2.3 任务详情
- 完整描述
- 子任务列表
- 更新历史
- 操作按钮（编辑/删除/完成）

### 3.3 设计风格

#### 3.3.1 视觉风格
- **主题**：深色模式为主，渐变色点缀
- **配色**：参考技能交换网站（紫蓝渐变）
- **图标**：虾的元素（🦞）
- **动画**：流畅的过渡效果

#### 3.3.2 交互体验
- 拖拽排序
- 快捷键支持
- 响应式设计（支持移动端）

## 4. 技术方案

### 4.1 技术栈
- **前端**：纯 HTML + CSS + JavaScript（避免构建复杂度）
- **存储**：GitHub（Markdown 文件 + JSON）
- **部署**：Cloudflare Pages
- **API**：Cloudflare Workers

### 4.2 数据结构

#### 4.2.1 任务数据（JSON）
```json
{
  "id": "task-001",
  "title": "实现微信自动回复",
  "description": "调研并实现微信自动回复功能",
  "category": "龙虾+微信",
  "priority": "P1",
  "status": "in-progress",
  "subtasks": [
    {
      "id": "subtask-001",
      "title": "调研可用方案",
      "completed": true
    }
  ],
  "createdAt": "2026-03-06T15:00:00Z",
  "updatedAt": "2026-03-06T15:30:00Z",
  "updatedBy": "丑丑虾"
}
```

#### 4.2.2 分类数据
```json
{
  "id": "cat-001",
  "name": "龙虾+微信",
  "color": "#4e54c8",
  "icon": "💬",
  "description": "探索龙虾与微信的集成应用"
}
```

### 4.3 文件结构
```
todolist-data/
├── categories.json          # 分类定义
├── tasks/
│   ├── cat-001/            # 按分类组织
│   │   ├── task-001.json
│   │   └── task-002.json
│   └── cat-002/
└── history/                # 更新历史
    └── 2026-03-06.json
```
