# 皮皮虾的 TodoList 项目

## 项目目标
创建一个愿望清单式的 todolist 系统，支持多 agent 协作更新，类似技能交换网站的逻辑。

## 需求拆解

### 一、龙虾 + 微信研究
**目标**：探索龙虾与微信的集成应用

- [ ] 1.1 实现微信自动回复功能
  - [ ] 1.1.1 调研可用的微信接入方案（WeFlow/企业微信/飞书）
  - [ ] 1.1.2 设计自动回复规则引擎
  - [ ] 1.1.3 实现基础自动回复
  - [ ] 1.1.4 测试并优化回复质量

- [ ] 1.2 微信朋友圈分析（多模态）
  - [ ] 1.2.1 获取朋友圈数据（图片+文字）
  - [ ] 1.2.2 设计多模态分析模型
  - [ ] 1.2.3 实现朋友圈内容分析
  - [ ] 1.2.4 生成分析报告

### 二、飞书 + 龙虾持续研究
**目标**：深度集成飞书官方能力

- [ ] 2.1 接入飞书新官方插件
  - [ ] 2.1.1 调研飞书官方插件权限
  - [ ] 2.1.2 申请并配置插件
  - [ ] 2.1.3 测试插件功能
  - [ ] 2.1.4 文档化配置流程

- [ ] 2.2 实现代替发送飞书消息
  - [ ] 2.2.1 研究飞书 API 权限
  - [ ] 2.2.2 设计消息代发逻辑
  - [ ] 2.2.3 实现代发功能
  - [ ] 2.2.4 测试并确保安全性

### 三、龙虾持续进化
**目标**：提升多 agent 协作效率

- [ ] 3.1 多 agents 高频使用场景落地
  - [ ] 3.1.1 分析当前 agents 能力（皮皮虾/乖乖虾/乐乐虾/锥锥虾）

  - [ ] 3.1.4 监控 agents 使用频率并优化

- [ ] 3.2 实现 todolist 网站
  - [ ] 3.2.1 需求文档编写（参考 clawhub/github 案例）
  - [ ] 3.2.2 设计 UI/UX（高质量、有调性）
  - [ ] 3.2.3 开发前端页面
  - [ ] 3.2.4 实现多 agent 协作更新逻辑
