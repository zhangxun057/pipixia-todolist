# 需求讨论记录

## 讨论 1：数据存储方案

**张洵**：用 GitHub 存储会不会太慢？每次更新都要 commit？

**丑丑虾**：
- GitHub 适合版本控制和多 agent 协作
- 可以批量更新，不是每个操作都 commit
- 本地先缓存，定期同步到 GitHub
- 或者用 Cloudflare KV 存储，GitHub 只做备份

**决策**：采用混合方案
- 主存储：Cloudflare KV（快速读写）
- 备份：GitHub（版本控制）
- 每小时自动同步一次

## 讨论 2：Agent 如何更新任务？

**张洵**：agents 怎么知道要更新哪些任务？

**丑丑虾**：
- 方案 1：通过 API 调用（需要 agent 主动）
- 方案 2：通过自然语言（"帮我把微信自动回复标记为完成"）
- 方案 3：定时检查 memory 文件，自动同步进度

**决策**：采用方案 2 + 方案 3
- 支持自然语言更新
- 自动从 memory 文件同步进度

## 讨论 3：UI 设计风格

**张洵**：要有调性，不要太普通。

**丑丑虾**：
- 参考 Linear（简洁、现代）
- 使用渐变色和微动画
- 虾的元素作为点缀
- 深色模式为主

**决策**：
- 主色调：紫蓝渐变（#4e54c8 → #8f94fb）
- 辅助色：橙色（#ff6b6b）表示优先级
- 字体：系统默认 + 等宽字体（代码）
- 图标：简约线条风格

## 讨论 4：移动端支持

**张洵**：我经常在手机上查看。

**丑丑虾**：
- 响应式设计必须做
- 移动端优先考虑查看，编辑可以简化
- 支持手势操作（滑动完成任务）

**决策**：完全响应式，移动端体验优先
